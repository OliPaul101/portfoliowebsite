Intro

Me and my team were tasked by our University, Norwich University of the Arts, to come up with a game idea, create a design document and a pitch for it, and then create a prototype, all in a week. So, we got thinking and came up with a Brewmaster. Brewmaster is a top down dungeon crawling adventure featuring challenging potion crafting combat.

My Designs

The image above is from the Brewmaster game pitch and is a good summary of what different enemies we have/ have planned for Brewmaster. After we decided about what enemies will be in the game, it was time for me to start designing their statistics (i.e health, weaknesses, damage, etc) and how they would interact with the world and player.

I wanted the enemies in Brewmaster to work together in groups so I had an idea that if the player walked into just one enemy's line of sight, that enemy would alert the rest and they would all swarm the player. I felt it best that the enemies swarmed the player in groups so the potion combat would feel more satisfying, as throwing potions that have AOE into a crowd of enemies would, I assume, be more fun and rewarding.

AI Code

Firstly, the perception component has to be setup. The bottom line adds a dynamic, essentially a function to execute if the perception component sees an actor. The function to execute is called 'OnPawnDetect'.

The 'OnPawnDetect' function has a parameter of an array of actors. Then there is a for loop, that loops through all the actors in that array, the actors in the array are actors that this perception component can see. During the loop, if the actor in the array is the players character, or 'AFiveStackPotionGameCharacter', then another function called 'AlertEnemies' is executed. Also, the blackboard value 'Player', is also set to the player character.

The AlertEnemies function essentially creates a radius around the enemy that has seen the player. This radius is set to collide specifically with pawns, as other enemies are just pawns.

After the radius is created, then enters a similar for loop used earlier that goes through the overlapping actors within the radius. If an overlapping actor is another enemy, then it executes a function on that enemy called 'chasePlayer'.

Chase player does, you guessed it, chase the player. Without having seen the player it sets the backboard value 'Player', like before, to the player character that the original enemy has seen.
