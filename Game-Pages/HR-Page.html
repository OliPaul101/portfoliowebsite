Intro

The Hardest Route was a game developed for the Global Game Jam 2023. The theme for the jam was 'ROOTS' so naturally we tried to think outside the box and came up with Hardest 'Route'. We were quite a small team with only five people (including myself). However, because of the small team, I found myself doing a lot more general jobs that I may have not done before. For example, I had to import all the assets, create animation blueprints, program the rooms, program the pick up, do the lighting for the level, etc.

Rooms Blueprint

This is the blueprint for the rooms I created that we used for Hardest Route. Each room had four doors and the doors had to be closed if the player was with an enemy and they had to open when the enemy was defeated, or when they ran out of stamina. Each room also had its own camera that the players perspective changed to whenever they went into a new room. Ideally there would be only one camera in use but because we only had 48 hours it was much faster for me to make it so that the players perspective would jump from one camera to the other, instead of one camera being used and making that jump to where I want.

There are also some variables in use as well. Most of these are simple checks to see if the player is still inside the room, if the player has finished the room or if the room is empty to begin with. But, there is also an integer that keeps track of how many enemies we spawned in the room. As we were making a game about finding the best route around the level, some rooms would have 3, 2 or possibly no enemies in at all. So it was important to keep track.

Above is a very simple blueprint function but I wanted to show how the Enemy Count integer was being used. Enemy Count was being used to not only keep track but also to check if a room was finished or not. Essentially we want the room doors to open when the enemies are defeated so whenever that happens to an enemy we decrement the enemy count. If the enemy count for that room then drops below 0 then the doors open and the player can proceed.

With the enemies tracked and counted, I would use this to update the doors in the room. It was quite simple, whenever the Update Doors function was called, it would see if the player was inside the room first. Then it would check to see if the enemy count is greater than 0. If not then the doors would stay shut, by making their Z location 0. If the enemies were defeated, i.e. the enemy counter is less than or equal to 0, then doors would open, by setting their z location to -500.
